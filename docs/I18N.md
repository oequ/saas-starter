# Internationalization (i18n)

Runtime localization for **angular-saas-starter-ui** uses [@jsverse/transloco](https://jsverse.github.io/transloco/) via `@oequ/i18n`. English is the only shipped locale today; adding another language does not require an app rebuild.

## Layout

| Path | Role |
|------|------|
| `libs/i18n/src/lib/provide-oequ-i18n.ts` | Registers Transloco, HTTP loader, `LOCALE_ID` |
| `libs/i18n/src/lib/locale-preference.service.ts` | Active language + `localStorage` (`oequ-locale`) |
| `libs/i18n/src/assets/i18n/<lang>/*.json` | Translation scopes (see loader `SCOPES` in `oequ-transloco-http-loader.ts`) |
| `apps/demo/project.json` | Copies `libs/i18n/src/assets/i18n` → `/i18n` at build time |

Demo app wiring (`apps/demo/src/app/app.config.ts`):

```ts
provideOequI18n(),
provideAppInitializer(() => inject(LocalePreferenceService).init()),
```

## Key convention

`scope.section.element` — scope matches the JSON file name:

- `shell.nav.billing` → `shell.json` → `{ "nav": { "billing": "…" } }`
- `auth.login.submit` → `auth.json`
- `onboarding.demo.title` → `onboarding.json`
- `org.usage.title` → `org-usage.json`
- `org.metrics.kpi.emailsSent` → `org-metrics.json`

**Loader scopes (English):** `common`, `shell`, `auth`, `onboarding`, `account`, `org-members`, `cookie`, `org-billing`, `paywall`, `org-usage`, `org-metrics`, `org-emails`, `org-api-keys`, `org-integrations`, `org-general`, `help`, `legal`, `errors`.

Scopes prefixed with `org-` are merged under `org.<segment>` (e.g. `org-members.json` → `org.members.*`, `org-api-keys.json` → `org.apiKeys.*`).

## Templates

```html
{{ 'shell.nav.metrics' | transloco }}
{{ 'auth.forgotPassword.sentStatus' | transloco: { email: submittedEmail() } }}
```

Import `TranslocoPipe` from `@oequ/i18n` in standalone components.

## TypeScript (toasts, services)

```ts
import { translateKey, translatePortError, portErrorToError, TranslocoService } from '@oequ/i18n';

// Inside injection context only:
toast.success(translateKey('shell.userMenu.help'));

// Port errors (async handlers — inject TranslocoService, not translateKey):
private readonly transloco = inject(TranslocoService);
toast.error(translatePortError(result.error, this.transloco));
throw portErrorToError(result.error, this.transloco);
```

## Port errors (`errors.json`)

Mock adapters return `PortError` with a **`reason`** key (and optional **`params`**) instead of English `message`. UI resolves copy via `translatePortError`:

1. `errors.reasons.<reason>` (with params)
2. `errors.codes.<code>` (fallback per `PortErrorCode`)
3. `error.message` (legacy)
4. `common.errorGeneric`

In adapters:

```ts
import { mockErr } from './mock-port-error';
return mockErr('VALIDATION', 'invalidInviteEmail');
return mockErr('CONFLICT', 'integrationAlreadyConnected', { name: catalogItem.name });
```

Keys live in `libs/i18n/src/assets/i18n/en/errors.json`. Helpers: `libs/i18n/src/lib/translate-port-error.ts`, `libs/adapters-mock/src/lib/mock-port-error.ts`.

## Activation config

`ActivationOnboardingConfig` / `ActivationDemoStepConfig` fields `title`, `subtitle`, `description`, and `actionLabel` are **translation keys**, not display strings. See `apps/demo/src/app/demo-activation.config.ts`.

## Add a locale (e.g. Deutsch)

1. Copy `libs/i18n/src/assets/i18n/en` → `de` and translate JSON files.
2. Register locale in `libs/i18n/src/lib/supported-locales.ts`:

   ```ts
   { id: 'de', labelKey: 'shell.locale.deutsch' }
   ```

3. Add label to `shell.json` (all languages): `"deutsch": "Deutsch"`.
4. For `DatePipe` / `CurrencyPipe`:

   ```ts
   import localeDe from '@angular/common/locales/de';
   registerLocaleData(localeDe);
   ```

5. Rebuild — add `<oequ-locale-switcher />` to the user menu (or your settings UI). The component is exported from `@oequ/i18n` but **not wired in the demo** until you ship a second locale; it enables the `<select>` when `SUPPORTED_LOCALES.length >= 2`.

## Migration checklist (per component)

1. Move strings into the correct scope JSON under `en/`.
2. Replace template text with `{{ 'scope.key' | transloco }}`.
3. Replace TS user-facing strings with `translateKey('scope.key')` where needed.
4. Keep E2E selectors on **English** copy until you add a dedicated locale job.
